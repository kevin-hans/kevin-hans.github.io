RedHat/Centos系OSにApacheの設定方法

apache centos redhat
ノウハウ
初回発表日:   2019-09-10

背景

昨今クラウドサービスを利用してシステム環境構築していくことが普通でしょう。さらにクラウドのサービス（例えばAWSの# Elastic Beanstalk）を利用すれば、コードをアップロードするだけアプリを簡単にデプロイできます。これでApache、Nginx、Passenger、IIS など使い慣れたサーバーの設定を省きます。
でも、もしApacheを面倒しなければならない場合どうすべきか。今はタイトルをトピックに説明しに行きます。

ソフトウェアバージョン

  • OS：CentOS-7
  • Apache: 2.4

タスク

  • セキュリティなApache環境を構築
  • HTTPS証明書
  • リバースプロキシモード

手順

1 アプリユーザーwwwの追加とパスワードの設定
1.1 ユーザー作成

useradd -s /sbin/nologin www

1.2 パスワード設定

useradd -s /sbin/nologin www

1.3 作成したユーザーの確認

cat /etc/passwd | grep www
id www

2 httpdパッケージのインストール
2.1 httpdパッケージの確認

yum list | grep httpd

2.2 httpdのインストール

yum -y install httpd httpd-tools httpd-devel httpd-manual

2.3 httpdインストール後の確認

yum list installed | grep httpd	   
httpd -version
systemctl start httpd
systemctl status http
ps -aux | grep httpd

2.4 httpd自動起動の設定

systemctl enable httpd
systemctl is-enabled httpd

3 httpd設置の変更
3.1 動作ユーザー・グループの変更
/etc/httpd/conf/httpd.conf 65〜66行目付近

User Apache
Group Apache
↓
User www
Group www

3.2 サーバー管理者のメールアドレス変更
/etc/httpd/conf/httpd.conf 86行付近

ServerAdmin root@localhost
↓
ServerAdmin username@domain

3.3 サーバーのホスト名変更
/etc/httpd/conf/httpd.conf 95行付近

#ServerName www.example.com:80
↓
ServerName app.domain.com:80

3.4 デフォルトコンテンツを無効化

cd /etc/httpd/conf.d/
mv welcome.conf welcome.conf.org

3.5 iconsフォルダを無効化

cd /etc/httpd/conf.d/
mv autoindex.conf autoindex.conf.org

3.6 ディレクトリ内容一覧表示を無効化
/etc/httpd/conf/httpd.conf
144行目付近

Options Indexes FollowSymLinks
↓
Options FollowSymLinks

3.7 HTTP TRACEメソッドのサポートを無効化
/etc/httpd/conf/httpd.confの尾に追加

#HTTP TRACE off
TraceEnable off

3.8 X-Frame-Optionsヘッダを設定
/etc/httpd/conf/httpd.confの尾に追加

Header append X-FRAME-OPTIONS “SAMEORIGIN”

3.9 apache バージョンの表示機能無効化
/etc/httpd/conf/httpd.confの尾に追加

ServerTokens ProductOnly
ServerSignature off

3.10 openssl脆弱性バージョンをインストールしていない確認

#heartbleedが存在するOpenSSLのバージョン
#OpenSSL 1.0.1 から 1.0.1f
#OpenSSL 1.0.2-beta から 1.0.2-beta1

yum info openssl

脆弱性バージョンであれば、バージョンアップする

yum install openssl

3.11 SSL/TLSにおける安全性の低い暗号スイートのサポート無効化
※ここ（3.11）以降４（4.1と4.2）まで新規ファイル/etc/httpd/conf.d/easyprovision.confに関わる。
全体的なイメージが下記になる。

SSLCipherSuite HIGH:!3DES:!aNULL:!MD5:!SEED:!IDEA:!RC4:!DH
SSLHonorCipherOrder on

3.12 ssl.confの設定変更SSLv2およびSSLv3無効化

SSLProtocol all -SSLv2 -SSLv3

3.13 SSL/TLSプロトコルのデータ圧縮無効化

SSLCompression off
※#Apache環境変数設定ファイル（/etc/sysconfig/httpd）に以下の設定追加（RedHat only）
export OPENSSL_NO_DEFAULT_ZLIB=1

3.14 SSL証明書とキーファイルの設置

#Server Certificate:
SSLCertificateFile /etc/pki/tls/certs/self-server.crt
#Server Private Key:
SSLCertificateKeyFile /etc/pki/tls/private/self-server.key

3.15 httpdのリバースプロキシの設定
まずproxyモジュールが効いていると確認

httpd -M | grep proxy

リバースプロキシを設定

ProxyRequests off
ProxyVia off 
<Proxy *>
     Require all granted
</Proxy>
ProxyPass /path http://123.456.789.100:9999/app-path
ProxyPassReverse /path http://123.456.789.100:9999/app-path

4 httpd再起動
4.1 上記3に修正／新規されたファイルを適用

chown root:root /etc/httpd/conf/httpd.conf
chown root:root /etc/httpd/conf.d/easyprovision.conf

4.2 ファイル内容が正しいかチェック
```bash
https -t

4.3 httpd再起動
```bash
systemctl restart httpd
systemctl status httpd
ps -aux | grep httpd
