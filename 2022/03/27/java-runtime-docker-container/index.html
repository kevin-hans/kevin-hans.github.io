Java Runtime docker image 作り方

Spring cloud data flow(batch only mode)を立ち上げることを例にして説明する

まずはDockerfileを作成

今回はすでにJarファイルを手に入れるので、jdkではなく、jreを構築する。

FROM openjdk:11-jre-slim
RUN mkdir -p /app
WORKDIR /app
COPY ./spring-cloud-dataflow-server-2.9.2.jar .

#Env
ENV SPRING_CLOUD_DATAFLOW_FEATURES_STREAMS_ENABLED=false
ENV SPRING_CLOUD_DATAFLOW_FEATURES_SCHEDULES_ENABLED=false
ENV SPRING_CLOUD_DATAFLOW_FEATURES_TASKS_ENABLED=true
ENV spring_datasource_url=jdbc:postgresql://host.docker.internal:5432/dbname
ENV spring_datasource_username=user
ENV spring_datasource_password=password
ENV spring_datasource_driverClassName=org.postgresql.Driver
ENV spring_datasource_initialization_mode=always

ENTRYPOINT ["java","-jar","/app/spring-cloud-dataflow-server-2.9.2.jar"]

build docker image

docker build . -t scdf

start docker container

Dockerfileのenv変数と異なる場合、下記のようにカスタマイズできる。

docker run --name=spring-cloud-data-flow -d -p 9393:9393 \
-e spring_datasource_url='jdbc:postgresql://xxx.xxx.xxx.xxx:5432/dbname?currentSchema=xxx' \
-e spring_datasource_username='xxx' \
-e spring_datasource_password='xxx' \
scdf
